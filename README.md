# 심플 메이플 MMO (Simple Maple MMO)

브라우저에서 동작하는 2D 횡스크롤 멀티플레이어 RPG (메이플스토리 스타일 간소화 버전).
**Node.js + Express + Socket.IO**(서버) / **Phaser 3**(클라이언트)로 구현합니다.

> 핵심 게임 루프: **이동 → 몬스터 사냥 → EXP 획득 → 레벨업**
> 아키텍처: **반(半)권위 모델** — 이동은 클라이언트 보고 + 서버 검증, 전투/성장은 서버 권위.
> 계정: **닉네임 + 비밀번호**(첫 접속 시 자동 가입) · 진행도는 서버에 **영구 저장**(재시작해도 유지).

## 요구 사항

- **Node.js LTS** (권장: v20 이상 — 개발용 `npm run dev` 의 `--watch` 사용)
- 최신 브라우저 (Chrome/Firefox/Edge 등)

설치 확인:

```bash
node -v
npm -v
```

## 설치 & 실행

```bash
# 1) 의존성 설치
cd maple-mmo
npm install

# 2) 서버 실행
npm start
#  또는 개발 모드(파일 변경 시 자동 재시작):
npm run dev
```

실행되면 콘솔에 아래와 같이 출력됩니다:

```
==============================================
  심플 메이플 MMO 서버 실행 중
  로컬:  http://localhost:3000
  LAN :  http://<서버IP>:3000  (같은 네트워크에서 접속)
==============================================
```

브라우저에서 **http://localhost:3000** 접속 → 닉네임 입력 → **시작**.

### LAN(같은 네트워크) 다중 접속

- 서버는 이미 `0.0.0.0`(모든 인터페이스)에 바인딩되어 있습니다.
- 서버 PC의 IP 확인:
  ```bash
  hostname -I
  ```
- 다른 기기에서 브라우저로 `http://<서버IP>:3000` 접속.
- 방화벽에서 3000 포트가 막혀 있다면 개방하세요(예: `sudo ufw allow 3000`).
- 포트 변경: `PORT=4000 npm start`

## 몬스터 종류

지상과 플랫폼 곳곳에 다양한 몬스터가 스폰됩니다(스탯은 `server/config.js` 의 `MONSTER_TYPES` 에서 튜닝).

| 종류 | HP | EXP | 접촉 데미지 | 특징 |
|------|----|-----|------------|------|
| 달팽이 | 15 | 5 | 3 | 가장 약하고 느림(입문용) |
| 초록슬라임 | 25 | 10 | 5 | 기본 몬스터 |
| 빨강슬라임 | 40 | 16 | 8 | 빠름 |
| 버섯 | 60 | 24 | 11 | 단단함 |
| 멧돼지 | 80 | 32 | 14 | 빠르고 아픔 |
| 돌골렘 | 140 | 60 | 20 | 느리지만 매우 단단한 미니보스 |

> 초기·리스폰 위치는 각자의 배회 범위 안에서 **무작위**로 결정됩니다(같은 자리 반복 방지).
> 몬스터의 위치·HP·데미지·리스폰은 **모두 서버에서 계산**하며, 클라이언트는 서버가 준
> 타입 정보(색/크기)로 텍스처를 만들어 결과만 렌더링합니다.

## 테스트 체크리스트

### Phase 0 — 셋업 확인
- [ ] `npm install` 이 오류 없이 완료된다.
- [ ] `npm start` 후 콘솔에 "서버 실행 중" 배너가 출력된다.
- [ ] `http://localhost:3000` 접속 시 닉네임 입력 화면이 보인다.
- [ ] 브라우저 콘솔(F12)에 `[net] 서버 연결됨: <소켓ID>` 가 찍힌다.
- [ ] 서버 콘솔에 `[연결] 클라이언트 접속: ...` / `[입장] ...` 로그가 찍힌다.
- [ ] `curl http://localhost:3000/health` 가 `{"ok":true,...}` 를 반환한다.

### Phase 1 — 싱글 이동 확인
- [ ] 닉네임 입력 후 캐릭터(파란 도형)가 맵에 스폰되어 바닥으로 떨어진다.
- [ ] `←`/`→`(또는 `A`/`D`)로 좌우 이동, 방향에 따라 캐릭터가 좌우로 뒤집힌다.
- [ ] `↑`/`W`/`Space`로 점프하고 중력으로 다시 내려온다.
- [ ] 공중의 발판(플랫폼) 위에 올라설 수 있다.
- [ ] 캐릭터가 화면 밖으로 나가면 카메라가 따라 이동한다(맵 경계에서 멈춤).
- [ ] 캐릭터 머리 위에 닉네임이 표시된다.

### Phase 2 — 멀티플레이어 동기화 확인 (브라우저 2개 이상)
- [ ] 브라우저 2개(또는 다른 기기)에서 각각 다른 닉네임으로 접속한다.
- [ ] 서로 상대 캐릭터(초록색)와 머리 위 닉네임이 보인다.
- [ ] 한쪽이 이동/점프하면 다른 쪽 화면에서 부드럽게(보간) 따라 움직인다.
- [ ] 우상단 "접속자: N명" 숫자가 접속/퇴장에 맞게 갱신된다.
- [ ] 한쪽이 탭을 닫으면 다른 쪽에서 해당 캐릭터가 사라진다.

### Phase 3 — 몬스터 & 전투 확인
- [ ] 맵 곳곳(지상·플랫폼)에 여러 종류의 몬스터가 다수 스폰되어 좌우로 배회한다.
- [ ] `X` 키 또는 마우스 좌클릭으로 공격하면 앞쪽에 슬래시 이펙트가 나온다.
- [ ] 몬스터를 때리면 피격 플래시 + 데미지 숫자가 뜨고 HP 바가 줄어든다.
- [ ] HP 0이 되면 몬스터가 사라지고 `+EXP` 가 표시되며 EXP가 증가한다(종류마다 EXP 다름).
- [ ] 잠시 후 배회 범위 안 **랜덤 위치**에서 리스폰된다(리스폰 시간은 종류마다 다름).
- [ ] 여러 명이 접속해 있어도 몬스터 HP/EXP가 일관되게 처리된다(서버 권위).

### Phase 4 — 성장 시스템 확인
- [ ] 좌하단에 HUD(레벨 / HP·MP·EXP 바)가 표시된다.
- [ ] 슬라임을 처치하면 EXP 바가 차오르고 `+EXP` 가 뜬다.
- [ ] EXP가 가득 차면 레벨업 → `LEVEL UP!` + 골든 링 연출, 레벨/최대HP·MP·공격력이 오르고 HP/MP가 완전 회복된다.
- [ ] 슬라임에 몸이 닿으면 HP 바가 줄고 데미지 숫자 + 화면 흔들림이 나온다(피격 후 잠깐 무적).
- [ ] HP가 0이 되면 "쓰러졌습니다! N초 후 부활" 오버레이가 뜨고 조작이 잠긴다.
- [ ] 대기 후 스폰 지점에서 부활하며 HP/MP가 가득 찬 상태로 조작이 재개된다.
- [ ] 다른 플레이어가 죽으면 화면에서 사라지고, 부활 시 스폰 지점에 다시 나타난다(서버 권위).

### Phase 5 — 데이터 저장 확인
- [ ] 로그인 화면에 닉네임 + 비밀번호 입력란이 있다.
- [ ] 새 닉네임+비번으로 접속하면 자동 가입되어 Lv.1로 시작한다.
- [ ] 레벨을 올린 뒤 접속을 끊고 **서버를 재시작**해도, 같은 닉+비번으로 재접속하면 레벨/EXP가 유지된다.
- [ ] 틀린 비밀번호로 같은 닉에 접속하면 "비밀번호가 올바르지 않습니다" 오류가 뜨고 입장되지 않는다.
- [ ] 이미 접속 중인 계정으로 다시 접속하면 "이미 접속 중인 계정입니다" 오류가 뜬다.
- [ ] `data/accounts.json` 파일이 생성되고, 비밀번호가 평문이 아닌 해시로 저장된다.

> 자동 검증: `node test/integration_persistence.js` (격리 포트/데이터로 로그인→레벨업→**재시작**→진행도 유지까지 확인)
> — 사전에 `npm install --no-save socket.io-client` 필요.

### 자동 스모크 테스트 (선택)

헤드리스 브라우저(Puppeteer)로 동작을 자동 검증할 수 있습니다.
Puppeteer는 용량이 커서 기본 의존성에는 포함하지 않았습니다.

```bash
# 서버를 먼저 실행해 둔 상태에서:
npm install --no-save puppeteer          # 최초 1회 (Chromium 다운로드)
node test/e2e_phase1.js                   # Phase 1: 이동/점프/착지 등 11개 항목
node test/e2e_phase2.js                   # Phase 2: 2인 접속/이동 보간/퇴장 6개 항목
node test/e2e_phase3.js                   # Phase 3: 몬스터 스폰/배회/전투/EXP 4개 항목

# 실서버에 사람이 접속 중이면 다른 포트로 격리해 테스트:
PORT=3010 npm start &                      # 격리 서버
BASE_URL=http://localhost:3010 node test/e2e_phase3.js
```

> Ubuntu에서 Chromium 실행에 필요한 시스템 라이브러리와 `unzip` 이 없으면
> 설치가 필요할 수 있습니다.

## 통신 프로토콜 (현재 구현)

- **C→S**
  - `join {nick, password}` — 닉네임+비밀번호로 참가 (신규면 자동 가입, 기존이면 인증 후 진행도 로드)
  - `move {x,y,vx,vy,dir,anim}` — 내 위치 보고 (20Hz, 변경 시에만)
  - `attack {dir}` — 근접 공격 (판정/데미지는 서버가 계산)
- **S→C**
  - `joinError {reason}` — 인증 실패(비번 오류/미입력/중복 접속) → 클라 로그인 화면에 표시
  - `init {selfId, self, map, monsterTypes, players, monsters, tuning}` — 접속 초기 상태(인증 성공). `monsterTypes` 는 타입별 이름/크기/색(클라 텍스처 생성용)
  - `playerJoined {id,nick,x,y,dir,...}` — 다른 플레이어 접속
  - `playerMoved {id,x,y,vx,vy,dir,anim}` — 다른 플레이어 이동(서버 검증 후 중계)
  - `playerLeft id` — 다른 플레이어 퇴장
  - `playerAttacked {id,dir}` — 다른 플레이어 공격 모션
  - `monstersUpdate [{id,type,x,y,dir,hp,maxHp,alive}]` — 서버 틱(20Hz)마다 몬스터 상태
  - `monsterHit {id,hp,dmg,by,x,y}` — 몬스터 피격
  - `monsterDied {id,by,expDrop,x,y}` — 몬스터 사망
  - `expGained {gain,level,hp,maxHp,mp,maxMp,atk,exp,expToNext}` — 처치자에게 EXP 지급(+최신 스탯)
  - `levelUp {level,hp,maxHp,mp,maxMp,atk,exp,expToNext}` — 레벨업(본인 스탯 갱신). `expToNext`는 만렙이면 `null`
  - `playerLevelUp {id,level}` — 다른 플레이어 레벨업(연출용)
  - `playerHurt {hp,maxHp,dmg}` — 몬스터 접촉 피격(본인 HUD 갱신용)
  - `playerDied {id,x,y,respawnMs}` — 플레이어 사망
  - `playerRespawned {id,x,y,hp,maxHp,mp,maxMp}` — 플레이어 부활(스폰 지점 복귀)

> 좌표는 정수로 반올림해 전송하여 트래픽을 최소화합니다.
> 이동은 서버가 검증(맵 경계 클램프 + 과속 거부)한 뒤 다른 클라이언트로 중계하며,
> 수신 측은 100ms 보간 지연으로 부드럽게 렌더링합니다.
> 몬스터 위치·HP, 데미지·EXP·레벨·플레이어 HP는 **모두 서버에서만 계산**하고 클라이언트는 결과만 렌더링합니다.
> 성장 곡선(EXP 필요량 = `EXP_BASE × level^EXP_POW`), 레벨업 스탯 증가, 몬스터 접촉 데미지·무적창·부활 대기 시간은
> 모두 `server/config.js`(`GROWTH`/`PLAYER`/`MONSTER`)에서 튜닝합니다.
> 이후 Phase에서 `chat`, `useSkill` 등이 추가됩니다.

  현재는 코드로 생성한 단색 도형(플레이스홀더)만 사용하며, 이후에도 직접 제작 또는
  CC0 등 오픈 라이선스 에셋만 사용합니다.
- **보안**: 클라이언트 입력은 신뢰하지 않습니다. 데미지/EXP 등은 서버에서만 계산하며,
  이동은 서버에서 검증합니다(Phase 2~).
