# 심플 메이플 MMO (Simple Maple MMO)

브라우저에서 동작하는 2D 횡스크롤 멀티플레이어 RPG (메이플스토리 스타일 간소화 버전).
**Node.js + Express + Socket.IO**(서버) / **Phaser 3**(클라이언트)로 구현합니다.

> 핵심 게임 루프: **이동 → 몬스터 사냥 → EXP 획득 → 레벨업**
> 아키텍처: **반(半)권위 모델** — 이동은 클라이언트 보고 + 서버 검증, 전투/성장은 서버 권위.

## 진행 현황 (Phase)

| Phase | 내용 | 상태 |
|------|------|------|
| 0 | 프로젝트 셋업 (구조, Express 정적 서버, Phaser 빈 씬, Socket.IO 연결) | ✅ 완료 |
| 1 | 싱글 이동 (중력/플랫폼/점프, 좌우 이동, 카메라 추적) | ✅ 완료 |
| 2 | 멀티플레이어 동기화 (접속/이동/퇴장, 닉네임, 위치 보간) | ✅ 완료 |
| 3 | 몬스터 & 전투 (스폰/배회/리스폰, 공격 판정, 서버 권위 HP, EXP 드롭) | ✅ 완료 |
| 4 | 성장 시스템 (EXP/레벨/스탯, 레벨업, HUD 연동) | ⬜ 예정 |

| 6 | (선택) 확장: 인벤토리/아이템, 다중 맵+포털, 스킬, 데이터 저장 | ⬜ 예정 |

## 요구 사항

- **Node.js LTS** (권장: v20 이상 — 개발용 `npm run dev` 의 `--watch` 사용)
- 최신 브라우저 (Chrome/Firefox/Edge 등)

설치 확인:

```bash
node -v
npm -v
```

## 설치 & 실행

```bash
# 1) 의존성 설치
cd maple-mmo
npm install

# 2) 서버 실행
npm start
#  또는 개발 모드(파일 변경 시 자동 재시작):
npm run dev
```

실행되면 콘솔에 아래와 같이 출력됩니다:

```
==============================================
  심플 메이플 MMO 서버 실행 중
  로컬:  http://localhost:3000
  LAN :  http://<서버IP>:3000  (같은 네트워크에서 접속)
==============================================
```

브라우저에서 **http://localhost:3000** 접속 → 닉네임 입력 → **시작**.

### LAN(같은 네트워크) 다중 접속

- 서버는 이미 `0.0.0.0`(모든 인터페이스)에 바인딩되어 있습니다.
- 서버 PC의 IP 확인:
  ```bash
  hostname -I
  ```
- 다른 기기에서 브라우저로 `http://<서버IP>:3000` 접속.
- 방화벽에서 3000 포트가 막혀 있다면 개방하세요(예: `sudo ufw allow 3000`).
- 포트 변경: `PORT=4000 npm start`

## 테스트 체크리스트

### Phase 0 — 셋업 확인
- [ ] `npm install` 이 오류 없이 완료된다.
- [ ] `npm start` 후 콘솔에 "서버 실행 중" 배너가 출력된다.
- [ ] `http://localhost:3000` 접속 시 닉네임 입력 화면이 보인다.
- [ ] 브라우저 콘솔(F12)에 `[net] 서버 연결됨: <소켓ID>` 가 찍힌다.
- [ ] 서버 콘솔에 `[연결] 클라이언트 접속: ...` / `[입장] ...` 로그가 찍힌다.
- [ ] `curl http://localhost:3000/health` 가 `{"ok":true,...}` 를 반환한다.

### Phase 1 — 싱글 이동 확인
- [ ] 닉네임 입력 후 캐릭터(파란 도형)가 맵에 스폰되어 바닥으로 떨어진다.
- [ ] `←`/`→`(또는 `A`/`D`)로 좌우 이동, 방향에 따라 캐릭터가 좌우로 뒤집힌다.
- [ ] `↑`/`W`/`Space`로 점프하고 중력으로 다시 내려온다.
- [ ] 공중의 발판(플랫폼) 위에 올라설 수 있다.
- [ ] 캐릭터가 화면 밖으로 나가면 카메라가 따라 이동한다(맵 경계에서 멈춤).
- [ ] 캐릭터 머리 위에 닉네임이 표시된다.

### Phase 2 — 멀티플레이어 동기화 확인 (브라우저 2개 이상)
- [ ] 브라우저 2개(또는 다른 기기)에서 각각 다른 닉네임으로 접속한다.
- [ ] 서로 상대 캐릭터(초록색)와 머리 위 닉네임이 보인다.
- [ ] 한쪽이 이동/점프하면 다른 쪽 화면에서 부드럽게(보간) 따라 움직인다.
- [ ] 우상단 "접속자: N명" 숫자가 접속/퇴장에 맞게 갱신된다.
- [ ] 한쪽이 탭을 닫으면 다른 쪽에서 해당 캐릭터가 사라진다.

### Phase 3 — 몬스터 & 전투 확인
- [ ] 맵에 슬라임(빨강) 3마리가 스폰되어 좌우로 배회한다.
- [ ] `X` 키로 공격하면 앞쪽에 슬래시 이펙트가 나온다.
- [ ] 슬라임을 때리면 피격 플래시 + 데미지 숫자가 뜨고 HP 바가 줄어든다.
- [ ] HP 0이 되면 슬라임이 사라지고 `+EXP` 가 표시되며 좌하단 EXP가 증가한다.
- [ ] 잠시 후(기본 4초) 같은 자리에서 슬라임이 리스폰된다.
- [ ] 여러 명이 접속해 있어도 몬스터 HP/EXP가 일관되게 처리된다(서버 권위).

### 자동 스모크 테스트 (선택)

헤드리스 브라우저(Puppeteer)로 동작을 자동 검증할 수 있습니다.
Puppeteer는 용량이 커서 기본 의존성에는 포함하지 않았습니다.

```bash
# 서버를 먼저 실행해 둔 상태에서:
npm install --no-save puppeteer          # 최초 1회 (Chromium 다운로드)
node test/e2e_phase1.js                   # Phase 1: 이동/점프/착지 등 11개 항목
node test/e2e_phase2.js                   # Phase 2: 2인 접속/이동 보간/퇴장 6개 항목
node test/e2e_phase3.js                   # Phase 3: 몬스터 스폰/배회/전투/EXP 4개 항목

# 실서버에 사람이 접속 중이면 다른 포트로 격리해 테스트:
PORT=3010 npm start &                      # 격리 서버
BASE_URL=http://localhost:3010 node test/e2e_phase3.js
```

> Ubuntu에서 Chromium 실행에 필요한 시스템 라이브러리와 `unzip` 이 없으면
> 설치가 필요할 수 있습니다.

## 통신 프로토콜 (현재 구현)

- **C→S**
  - `join {nick}` — 닉네임으로 참가
  - `move {x,y,vx,vy,dir,anim}` — 내 위치 보고 (20Hz, 변경 시에만)
  - `attack {dir}` — 근접 공격 (판정/데미지는 서버가 계산)
- **S→C**
  - `init {selfId, self, map, players, monsters, tuning}` — 접속 초기 상태
  - `playerJoined {id,nick,x,y,dir,...}` — 다른 플레이어 접속
  - `playerMoved {id,x,y,vx,vy,dir,anim}` — 다른 플레이어 이동(서버 검증 후 중계)
  - `playerLeft id` — 다른 플레이어 퇴장
  - `playerAttacked {id,dir}` — 다른 플레이어 공격 모션
  - `monstersUpdate [{id,x,y,dir,hp,maxHp,alive}]` — 서버 틱(20Hz)마다 몬스터 상태
  - `monsterHit {id,hp,dmg,by,x,y}` — 몬스터 피격
  - `monsterDied {id,by,expDrop,x,y}` — 몬스터 사망
  - `expGained {exp,total}` — 처치자에게 EXP 지급

> 좌표는 정수로 반올림해 전송하여 트래픽을 최소화합니다.
> 이동은 서버가 검증(맵 경계 클램프 + 과속 거부)한 뒤 다른 클라이언트로 중계하며,
> 수신 측은 100ms 보간 지연으로 부드럽게 렌더링합니다.
> 몬스터 위치·HP, 데미지·EXP는 **모두 서버에서만 계산**하고 클라이언트는 결과만 렌더링합니다.
> 이후 Phase에서 `levelUp`, `chat` 등이 추가됩니다.

## 주의사항

- ⚠️ **에셋 저작권**: 실제 메이플스토리 스프라이트/리소스는 사용하지 않습니다.
  현재는 코드로 생성한 단색 도형(플레이스홀더)만 사용하며, 이후에도 직접 제작 또는
  CC0 등 오픈 라이선스 에셋만 사용합니다.
- **보안**: 클라이언트 입력은 신뢰하지 않습니다. 데미지/EXP 등은 서버에서만 계산하며,
  이동은 서버에서 검증합니다(Phase 2~).
